# Day 19 - JVM 内存与垃圾回收总结

## 一、内存问题核心概念

### 1. 内存溢出（Out Of Memory, OOM）
- **定义**：程序在申请内存时，没有足够的内存空间供其使用。
- **常见场景**：
  - 堆空间不足（`Java heap space`）
  - 方法区/元空间不足（`Metaspace`）
  - 直接内存溢出（`Direct buffer memory`）

### 2. 内存泄漏（Memory Leak）
- **定义**：对象已经不再使用，但仍然被引用，导致 GC 无法回收。
- **本质区别**：
  - 泄漏是**原因**
  - 溢出是**结果**

### 3. 内存溢出 vs 内存泄漏
| 对比项 | 内存泄漏 | 内存溢出 |
|--------|----------|----------|
| 是否还能使用 | 可以运行但越来越慢 | 直接崩溃 |
| 是否一定发生 | 不一定 | 一定发生 |
| 本质 | 无用对象未释放 | 内存不足 |

---

## 二、内存泄漏产生原因

- 静态集合类持有对象引用
- 未关闭资源（数据库连接、IO流）
- 监听器未移除
- ThreadLocal 使用不当
- 缓存使用不当（无限增长）

---

## 三、栈溢出（Stack Overflow）

### 发生条件：
- 方法递归调用过深
- 栈帧不断压入但没有释放

### 示例：
```java
public void test() {
    test(); // 无限递归
}
````

---

## 四、垃圾回收（GC）基础

### 1. JVM 垃圾回收的目标

* 回收不再使用的对象
* 自动管理内存生命周期

---

### 2. 如何判断对象“死亡”

#### 方法一：引用计数法（了解）

* 每个对象有引用计数
* 缺点：无法解决循环引用

#### 方法二：可达性分析（主流）

* 从 **GC Roots** 出发
* 无法到达的对象即为垃圾

---

### 3. GC Roots 包括：

* 栈中的局部变量
* 方法区中的静态变量
* 常量引用
* 本地方法引用（JNI）

---

## 五、垃圾回收算法

### 1. 标记-清除（Mark-Sweep）

* 标记垃圾 → 清除
* 缺点：内存碎片

### 2. 复制算法（Copying）

* 新生代使用
* 优点：无碎片
* 缺点：空间利用率低

### 3. 标记-整理（Mark-Compact）

* 老年代使用
* 优点：无碎片
* 缺点：移动对象成本高

---

## 六、分代收集思想

JVM 将内存分为：

### 1. 新生代（Young Generation）

* Eden + Survivor区
* 使用复制算法
* GC频繁（Minor GC）

### 2. 老年代（Old Generation）

* 存放长期存活对象
* 使用标记-整理
* GC较少（Major GC / Full GC）

---

## 七、GC 类型

| 类型       | 说明        |
| -------- | --------- |
| Minor GC | 新生代回收     |
| Major GC | 老年代回收     |
| Full GC  | 整个堆 + 方法区 |

---

## 八、GC 触发条件

### Young GC：

* Eden 区满

### Full GC：

* 老年代空间不足
* System.gc()
* Metaspace不足
* 晋升失败（Promotion Failure）

---

## 九、常见垃圾回收器

### 1. Serial

* 单线程
* Stop-The-World

### 2. Parallel（吞吐量优先）

* 多线程
* 适合后台计算

### 3. CMS（Concurrent Mark Sweep）

* 低停顿
* 缺点：内存碎片、CPU占用高

### 4. G1（Garbage First）

* 面向服务端
* 可预测停顿
* 按 Region 管理内存

---

## 十、CMS vs G1

| 特性   | CMS | G1     |
| ---- | --- | ------ |
| 内存结构 | 连续  | Region |
| 是否压缩 | 否   | 是      |
| 停顿控制 | 较弱  | 强      |
| 推荐程度 | 已过时 | 推荐使用   |

---

## 十一、垃圾收集器选择建议

* 小型应用：Serial / Parallel
* 高并发低延迟：G1（主流选择）
* 避免使用：CMS（逐渐淘汰）

---

## 十二、JVM 调优思路（入门）

* 合理设置堆大小（`-Xms -Xmx`）
* 调整新生代比例（`-XX:NewRatio`）
* 选择合适 GC（推荐 G1）
* 监控 GC 日志分析性能瓶颈

---

## 总结

本阶段核心掌握：

* 内存问题三件套：**泄漏 / 溢出 / 栈溢出**
* GC 核心机制：**可达性分析 + 分代收集**
* 三大算法：**标记清除 / 复制 / 整理**
* 常见 GC：**CMS vs G1**
* 调优方向：**参数 + GC选择 + 监控**

👉 重点理解：
**为什么会发生问题（原理） > 怎么解决（手段）**
